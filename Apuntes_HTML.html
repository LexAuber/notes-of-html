Las BLOCK se les tiene que forzar a tener etiquetas dentro de ellas mediante CSS a ser posible.

Todas las etiquetas empiezan con < y acaban con />. Exceptuando algunas como Link o Meta por ejemplo, que no necesitan cierre, porque las mismas etiquetas son el contenido. Se pueden cerrar igualmente dentro de la misma con "/". En todo momento las etiquetas se escribiran en minuscula. Solo el contenido en ellas se puede escribir en mayuscula. En el Inicio del documento siempre tendremos que poner la siguiente linea de codigo:

<!DOCTYPE html>

Lo que realizamos en esta linea de codigo, es una declaracion del tipo de documento de html que vamos a usar, en este caso HTML5.

Partes esenciales en la estructura de HTML:

HTML va a ser la primera de las etiquetas que vamos a poner, justo despues de la declaracion. Tiene que estar antes que HEAD y BODY, ya que conforma toda la forma del documento de HTML.

HEAD, es la cabecera del documento(parte invisible), se cierra siempre con />, se utiliza mas para la funcionalidad de la pagina, normalmente no se vera el contenido ya que es solo para el navegador, pero si se desea se puede hacer visible. Tambien se utiliza para poder añadir CSS en la misma pagina, aunque normalmente se suele externalizar via archivo externo. Las etiquetas que van dentro de head: title, style, meta, link, script, noscript, base.

  • En Head se pondran varias etiquetas METAS, una para referirse al CHARacter SETup, que es el formato de codificacion de caracteres, basicamente para que no se ignoren los simbolos de pronunciacion, o de escritura, acentos, interrogantes, etc... La forma en que se pone es esta:

    • <meta charset="UTF-8">

  • Otros elementos que se utilizan en HEAD con META, es el primer paso para la web responsive. Aunque esto normalmente lo hace CSS, ayuda al navegador que se utilize, a adaptar la pagina web al tipo de pantalla, y sus dimensiones, y que se vea bien al final. La linea de codigo es la siguiente:

    • <meta name="viewport" content="width-device-width, initial-scale=1.0">

  • Hay mas etiquetas y propiedades con META para ayudar a adaptar la pagina al buscador, o navegador, como por ejemplo, puede utilizarse para que tenga funcionalidad en Internet Explorer, aunque hoy en dia se utiliza menos y el soporte del mismo esta retirado. Tambien serviran para otros navegadores, Firefox, Edge, Opera, Safari...etc

TITLE: Define el titulo de la pagina, sirve para indicarlo al navegador, para los motores de busqueda e indexarlo. Debe de ser coherente.

SCRIPT: Se usa para incoporar scripts mediante el atributo "src". Lo que suelen hacer son manipulacion de imagenes, validacion de datos, canvios dinamicos de documentos, etc...

NOSCRIPT: Aparece un mensaje en el caso de que el navegador tenga desactivadas las funciones de javascript

STYLE: Sirve para añadir una hora de estilos en el head. Ya puede ser interna o externa, si es externa debera tener la etiqueta link, con los atributos rel, type, y href.

META: Sirve para añadir informacion adicional. Tiene atributos: charset, description, keywords, author, y refresh.

DESCRIPTION: Cuanto mas corto mejor, si no los buscadores pueden penalizarte.

BASE: Especifica la URL base para todos los enlaces con ruta relativa. Sirve para indicar una ruta base, y el resto de links de la pagina partiran de esa base.

BODY, es el cuerpo del documento, se cierra siempre y se utiliza para la parte visual de la pagina, ya sean, frases, tablas, u otras cosas de caracter informativo.

Etiquetas de HTML:

La estructura de una etiqueta (tag) es, < Etiqueta -> Atributo -> Valor />. En maquetacion hay que saber que elementos, atributos o etiquetas son inline y cuales block. Las etiquetas sin cierre, son tags que no tienen contenido mas que los atributos añadidos. Se puede cerrar igualmente con "/", un ejemplo es la etiqueta IMG, que seria <img src="x"/>

Etiqueta P, sirve para Paragrafos, se cierra siempre, se debe seguir una etica con los paragrafos. Es una etiqueta tipo BLOCK.

Etiquetas <!-- con --> se puede escribir comentarios utiles en la pagina de programacion.

Etiquetas como EM sirven para enfatizar, la I sirve para poner de estilo italics, la STRONG sirve para enfatizar algo mucho mas importante y dar importancia semantica, la B o bolt, tiene la misma funcion que Strong pero sin importancia semantica.

Etiquetas META sirve para añadir informacion extra de la pagina, tanto para el navegador como para el motor de busqueda (SEO).

Etiqueta LINK, es una vinculacion a un archivo de estilo CSS externo.

Etiqueta SCRIPT, es una vinculacion a un archivo Javascript externo.

Etiqueta BR o Brake, sirve para forzar un salto de linea.

Etiqueta HR realiza un corte visual, para el diseño, es mejor CSS.

Etiqueta A o Hipervinculo, enlaza a una pagina web o una pagina dentro de nuestro indice (Etiqueta INLINE), se tiene que poner contenido para que aparezca el enlace, ej: "<a href="x.jpg"/>TEXTO ENLACE</a>

Etiqueta IMG se utiliza para imagenes, y usa el tag SRC para enlazar las imagenes, no necesita un cierre "/".

Etiqueta ABBR, o abrebiation, sirve para las abreviaciones o siglas, como WHO (World Health Organitzation)

Etiqueta Blockquote sirve para las citas, con el atributo CITE, el enlace de la cita, y luego la cita en si.

Etiqueta FIGURE usada para definir una imagen que no es indispensable, es opcional, pero refuerza el contenido. Añade unos margenes

Etiqueta FIGCAPTION añade un pie de imagen, texto de la imagen descriptiva

Etiqueta DETAILS, permite trabajar con la etiqueta SUMARY, que es el sumario en formato lista, esconde lo que pongamos dentro o mostrarlo, son detalles adicionales. Puedes añadir paragrafos con el nombre de titulos de libros por ejemplo.

Etiqueta SUMMARY define una cabezera visible para el elemento DETAILS, se pone dentro de DETAILS

Etiqueta SPAN no tiene un significado especial, se usa para dar estilo a partes diferentes del texto con CSS. Es un contenedor de linea, por lo que se puede usar como contenedor de texto, y si puede ser, sera el ultimo recurso por posibles problemas de modificaciones a posteriori.

Etiqueta MARK Resalta texto

Etiqueta TIME define una fecha o hora

Etiqueta FORM es para formularios.

Etiqueta FIGURE convierte una imagen en un bloque

Etiqueta DIV, es una division en el que podemos poner cualquier cosa. Es un elemento a nivel de bloque que puede ser usado como contener para agrupar otros elementos html. Sirven para aplicar estilos, definir capas y distribuir los elementos de la pagina. Tradicionalmente se utiliza para maquetar paginas y aplicar efectos de Javascript.

Etiquetas Semanticas (parecidas al DIV):

En HTML5 parte de la faena de maquetacion pasa por nuevas etiquetas semanticas que veremos mas adelante, como HEADER, FOOTER,NAV, ARTICLE, ASIDE, FIGURE, o SECTION.

Dentro de SECTION (Puede ser por ejemplo seccion de noticias) se puede poner la etiqueta ARTICLE que sera el contenido en concreto(En la seccion de noticias, serian las noticias en si mismas), los ASIDE (fuera de SECTION) puede estar en cualquier parte de la pagina pero suele ser informacion adicional o extra, o externa, Tops, o cosas similares. En el FOOTER, suelen ser datos de contacto, no se recomienda ponerlo dentro de SECTION.

Etiqueta MAIN, es parecido al BODY, es la que tiene toda la informacion de la pagina, que no es lo mismo que todo el contenido visual de la web entera. Dentro de el, puede estar la parte central de la pagina SECTION, ARTICLE, ASIDE, pero no estaran ni HEADER ni FOOTER.

  • HEADER -> cabecera
  • MAIN -> cuerpo
  • FOOTER ->pie de pagina

Tipos de listas

Etiqueta UL, sirve para crear listas sin orden (Unordened List). Existen atributos o propiedades para cambiar la forma de listado, por defecto aparecen redondas, pero pueden ser estilizadas via CSS, y cambiar la forma por cuadrados, redondas huecas, etc...

Etiqueta OL, sirve para crear listas (Ordened List). Existen atributos o propiedades para cambiar la numeracion, por numeros, letras, numeros romanos,etc... Se escriben junto al atributo/propiedad TYPE. El atributo/propiedad REVERSED indica que sigue numeracion de abajo a arriba.

Dentro de OL y UL estan:

  • Sub-Etiqueta LI Objeto o Item en una lista(List Item).

Etiqueta DL lista de descripciones (Description List).

  • Sub-Etiqueta DT lista de descripcion de terminos(Description Terms).

  • Sub-Etiqueta DD lista de descripcion de definiciones(Description Definition).

Las listas se pueden anidar dentro de las mismas, tener listas dentro de listas.

Etiquetas TABLE, inicio de una tabla

  • Etiqueta TR, define una fila de la tabla
  • Etiqueta TD, define una unica celda de la tabla
  • Etiqueta TH define la capçalera de la tabla
  • Etiqueta CAPTION, define la leyenda de la tabla
  • Etiqueta COLGROUP, especifica un grupo de columnas, para darle estilo despues en CSS
    • Etiqueta COL, especifica las propiedades de una columna dentro de la etiqueta COLGROUP
  • Etiqueta THEAD, agrupa el contenido de la cabecera de la tabla, util para dar formatos
  • Etiqueta TBODY, agrupa el contenido de una tabla, util para dar formatos
  • Etiqueta TFOOT,agrupa el contenido del pie de la tabla, util para dar formatos
  • Etiquetas COLSPAN, indica los espacios en celdas que ocupara un elemento en horizontal, para que sea efectivo, se tienen que crear las celdas necesarias iguales al COLSPAN
  • Etiqueta ROWSPAN indica los espacios en celdas que ocupara un elemento en vertical. Misma regla de COLSPAN.

    Otras cosas de HTML

    La estructura de las carpetas es, para imagenes ( IMG, IMAGES, IMATGES, IMAGEN ), para estilos ( CSS, ESTILOS, STYLES ), para scripts o frameworks ( SCRIPTS, JS ), y para recursos externos ( RECURSOS, RESOURCES).

    Para hacer saltos o crear un Indice en la propia pagina, se utiliza el atributo ID, que identificara para realizar enlaces en la misma pagina. Se utiliza con los hipervinculos, para saltar a un mismo punto del documento. ID es un identificador unico, no puede repetirse dos veces, el CLASS si puede. Se utiliza el elemento # para hacer referencia al ID que queremos ir. EJ: <a href="#x.html">X</a> rigira a esa parte de la pagina.

    Se puede crear enlaces especiales como por ejemplo, a emails, para cuando queramos contactar aunque ya no se utiliza para evitar bots de spam y obliga al usuario a tener un gestor de mails instalado. ES RECOMENDABLE NO HACERLO, pero saber que existe. Se le puede poner el atributo SUBJECT para poner en el asunto del Email automaticamente. ej: "x@x.com?subject=Error"

    Se puede hacer enlaces a archivos para ejecutarlos, recomendado solo en intranets, es muy poco fiable y poco seguro en internet. Simplemente se añade la ruta relativa del .exe.

    Intentemos evitar estas etiquetas ID y CLASS, dentro de lo posible para tener un CSS menos problematico. Cuanto menos se concrete y mas se pueda utilizar, o reutilizar etiquetas mas generales, mejor.

    El microdata ayuda a dar informacion semantica en las estructuras

    La etiqueta TABLE, no se recomienda, es una texnica antigua, para ello es mejor los DIVS. Texnica dominante actualmente, mas versatil y potenten combinada con CSS, permite cambiar la distribucion de elementos, es mas amigable con los buscadores y navegadores aurales.

    • Consideraciones: Los elementos de bloque pueden contener elementos en linea o otros elementos en bloque. Los elementos en linea pueden conener datos (texto) o otros elementos en linea. Algunos elementos de bloque como los paragrafos pueden contnener directamente datos. Es un error colocar elementos de bloque dentro de elementos en linea y, aunque funcione, el validador HTML no lo pasara.
    • De momento, para trabajar, solamente tendremos en cuenta unas reglas basicas que son:
      • Los elementos de bloque se situan en lineas diferentes y cojen todo el amplio disponible
      • Los elementos de linea se situan en la misma linea, una tras de otra
      • Para hacer columnas, colocaremos diveros elementos de bloque seguidos, modificandolos por CSS

    Tipos de rutas para vinculos:

    Ruta relativa: Para las rutas de enlace a traves de HTML, tendremos que escribir el camino a seguir, ejemplo: Carpeta de imagen/imagen.jpg. Lo mismo con archivos CSS o JS.

    Ruta absoluta: Se tiene que escribir todo la direccion del archivo, ej: C:/Windows/Users/X/Desktop/Imagenes/y.jpg. Para rutas en un servidor se utiliza las dos barras, antes y despues de la carpeta /Imagenes/.

    Para llegar a otras carpetas, se tiene que salir de la carpeta actual, entrar a la otra y buscar el archivo, para ello se escribe ../ para ordenar la salida de la carpeta. Ej: ../imagenes/imagen.jpg.

    Codificacion:

    ASCII el primer codigo de todo., ANSI, e ISO-8859-1. Hoy en dia se trabaja con UTF-8. Las codificaciones tienen la mayoria de simbolos y demas, en caso de caracteres muy especiales, por ejemplos €, deberia escribirse "&euro_;" (sin barra baja). En HTML entities hay una lista de los posibles iconos donde encontramos el codigo para ponerlo. Suele tener al principio "&".

    Tipografias:

    Las serif, suelen ser la familias de tipografias mas predeterminadas. Serif sifnifica que la letra tiene "pie". Sans-serif es la letra sin el pie. El navegador usara la primera fuente de la lista especificada en "font-family" que tenga en el ordenador del usuario.

    Se pueden poner tipografias de otros tipos, siempre respetando la autoria.

    Font-face sirve para incorporar tipografias. Hay diferentes formatos para incoporar tipografias: TTF/OTF, WOFF(W3C), WOFF, SVG,... En CSS se indica en font family: myFont, y luego especificar la url con src:url().

    font-size hay diferentes formas de definir la medida. Absoluta: xx-small, small, medium, large.... Relativa: larger, smaller,... o por medida: pixeles, pulgadas, o porcentaje que se adaptara siempre.

    Como consejo es mejor definir el elemento raix con una medida en px, y luego el resto de unidades relativas con rem, o em.

    Font-style, estan los valores normal, italic o obliqua(forzar la cursiva).

    Font-weight podemos definir el grueso de la fuente, desde 100 a 900, siendo 400 la normal, 700 bold. Bolder mas negrita, lighter menos negrita.

    Font-variant sirve para que las letras minusculas parezcan mayusculas. Normal o Small-caps.

    Line-height modificamos la altura de las lineas entre ellas. Se pueden usar con porcentaje o unidades, y se puede utilizar para centrar las letras.

    FONT, define abreviadamente los atributos. Style, varian, wight, size/line-height o family. Se tiene que separar con espacios entre atributos. Siguiendo este orden o desordenado: style, variant, weight, size/line, family

    Color, se codifica con rgb, hexagesimal y nombre. Con rgb se puede hacer transparencia

    Text-align: valores left, right, justify o center.

    Text-indent: Es el espacio de inicio de la primera fase. Separa del margen la frase con medidas

    Text-decoration, decoracion del texto: none: nada, underline: subrallado, overline: linea por encima, y line-throught: rallado.

    Text-transform: transforma las palabras en diferentes medidas.

    Text-shadow: con valores: h-shadow, v-shadow, color, none y blur-radius. EJ: text-shadow: 2px 2px black, 0 0 20px blue.

    Letter-spacing: separacion de letras.

    Word-spacing: separacion de palabras

    Word-break: romper palabras entre dos letras con: normal, keep all o break all.

    White-space: maneja un espacio en blanco.

    Overflow: controla la visualizacion del texto en su contenedor, visible: sale de su encuadre, hidden: no sale de su encuadre, scroll: puedes bajar hasta el final del texto, auto: todo automaticamente. Solo funciona con DIV.

    Direcction: Especifica la direccion de escritura, se utiliza conjuntamente con "unicode-bidi", y sus valores son ltr(default) de Izq a Der, y rtl de Der a Izq.

    Vertical-align: Permite alinear en una misma linea, varios elementos del tipo inline, imagenes o textos por ejemplo. No se utiliza para centrar verticalmente un elemento dentro de un contenedor block.

    Para centrar con div, hay que entrar en utilizacion de diferentes elementos para alinear dentro de elementos en bloque. Line-height u overflow por ejemplo.

    Border, se utiliza para resaltar los bordes de un contenedor, se utiliza sobre elementos de bloque como div, o table. Se puede modificar todo, desde color, grueso, tipo, bordes, etc... se utiliza el border-style para concretar los tipos que queramos utilizar. Para redondear las puntas se utiliza border-radius y con unidades de medida.
